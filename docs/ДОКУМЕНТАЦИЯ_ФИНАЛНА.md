## 4. РЕЗЮМЕ

### 4.1. Цели и анализ

**Предназначение:** Уеб-платформа, свързваща пациенти, терапевти и администратори чрез инструменти за самопомощ и професионална подкрепа.

**Проблем:** Липса на достъпни дигитални инструменти за мониторинг на психичното здраве, локализирани за българския пазар.

**Решение:** MindLink+ предлага българска локализация, GDPR съответствие и верификация на специалисти чрез БЛС регистър.

### 4.2. Етапи на реализация

1. **Планиране:** Дефиниране на изисквания и архитектура.
2. **Backend:** Разработка с Node.js, Express и PostgreSQL.
3. **Frontend:** Интерфейс с React 18 и Material-UI.
4. **DevOps:** Контейнеризация с Docker и сигурен deployment.

### 4.3. Технологичен стек и сложност

**Предизвикателства:** RBAC (Role-Based Access Control) за 3 нива потребители, real-time чат с индикатори за писане, web scraping за верификация на лекари и сложна агрегирана аналитика.

**Стек:** React, Node.js, PostgreSQL, Docker, Nginx, Cloudflare.

## 5. ЛОГИЧЕСКО И ФУНКЦИОНАЛНО ОПИСАНИЕ

### 5.1. Архитектура

Платформата използва клиент-сървър архитектура, разделена на независими слоеве за по-добра мащабируемост и сигурност.

**[МЯСТО ЗА СНИМКА: АРХИТЕКТУРНА ДИАГРАМА]**

### 5.2. Модули

- **Auth:** Сигурна JWT автентификация.
- **Mood & Journal:** Модули за проследяване на емоции и водене на дневник.
- **Therapist System:** Управление на пациенти, споделяне на бележки и статистика.
- **Real-time Chat:** Комуникация в реално време.
- **Admin Panel:** Пълен контрол върху системата и потребителите.

### 5.3. База данни (ERD)

Релационен модел в PostgreSQL, включващ таблици за потребители, съобщения, записи за настроение (mood_entries) и релации пациент-терапевт.

## 6. РЕАЛИЗАЦИЯ И ДЕПЛОЙМЪНТ

### 6.1. Алгоритми и логика

**Аналитика:** Изчисляване на активност чрез формулата:

```
activePercentage = (activeUsers / totalAccounts) × 100
```

**Сигурност:** Паролите се хешират с Argon2/BCrypt; трафикът е защитен с TLS 1.3.

### 6.2. Инструкции за стартиране (Development)

```bash
# Backend setup
cd backend && npm install
echo "DATABASE_URL=postgresql://user:pass@localhost:5432/db" > .env
npm start

# Frontend setup
cd frontend && npm install
npm run dev
```

**[МЯСТО ЗА СНИМКА: СКРИЙНШОТ ОТ КОНЗОЛА – BACKEND/FRONTEND RUNNING]**

### 6.3. Docker и Production Deployment

За продукционна среда се използва docker-compose, който оркестрира трите основни услуги: frontend, backend и db.

**Стъпки за внедряване:**
1. Контейнеризация на микросервизите.
2. Настройка на Cloudflare Tunnel за заобикаляне на NAT и защитен достъп.
3. Автоматизиран ежедневен backup чрез cron задача.
